# FoodmoleGPT 训练报告

> 食品科学领域大语言模型 — 消融实验训练报告
>
> 更新日期：2026-03-10

---

## 1. 项目目标

基于开源基座模型，通过**领域继续预训练（CPT）**和**指令微调（SFT）**构建食品科学领域专用 LLM，并通过消融实验对比不同训练策略的效果。

### 消融实验设计

| 条件编号 | 训练策略 | 说明 |
|---------|---------|------|
| ① | Base（无训练） | 基线，直接用原始基座模型评测 |
| ② | SFT-Only | 仅做指令微调 |
| ③ | CPT-Only | 仅做领域继续预训练 |
| ④ | CPT → SFT | 先领域预训练，再指令微调（完整流水线） |

每个条件分别在 **Qwen3-8B-Base** 和 **Llama 3.1 8B** 两个基座上执行，共 **8 组实验**。

---

## 2. 基座模型

| 模型 | 参数量 | 特点 | 选用原因 |
|------|-------|------|---------|
| Qwen3-8B-Base | 8B | 阿里开源，中英双语能力强 | 中文食品科学场景适配 |
| Meta-Llama-3.1-8B | 8B | Meta 开源，英文能力强 | 跨模型对照，验证方法通用性 |

选择 8B 规模的理由：单卡 H200（141GB 显存）可完整加载，适合 LoRA 微调；比小模型（1-3B）知识容量更大；比大模型（70B+）训练成本低，适合消融实验。

## 3. 训练方法

### 3.1 LoRA（Low-Rank Adaptation）

所有实验统一使用 LoRA 微调，冻结基座模型权重，仅训练低秩适配器。

| 参数 | 值 | 说明 |
|------|-----|------|
| lora_rank | 64 | 低秩矩阵的秩，8B 模型的社区经验值 |
| lora_alpha | 128 | 缩放系数，alpha/rank = 2（标准比例） |
| lora_target | all | 对所有线性层加 LoRA（非仅 attention） |
| 可训练参数占比 | ~1-2% | 约 100-200M 参数 |

**为什么用 LoRA 而非全参微调（FFT）：**
- 显存效率高（~20-30GB vs FFT 的 ~60GB）
- 消融实验需跑多组，LoRA 节省 GPU 时间预算
- 学术研究表明 LoRA 在领域适配任务上接近 FFT 效果

### 3.2 CPT（Continual Pre-Training）— 领域知识注入

用食品科学领域语料继续预训练基座模型，使模型学习领域术语、概念和推理模式。

| 参数 | 值 |
|------|-----|
| 训练阶段 | pre-training（无监督，自回归语言建模） |
| 学习率 | 5e-5（较低，避免灾难性遗忘） |
| epochs | 1 |
| 学习率调度 | cosine decay |
| warmup 比例 | 1% |
| 序列长度 | 2048 tokens |
| 精度 | bf16 |

### 3.3 SFT（Supervised Fine-Tuning）— 指令对齐

用食品科学领域的指令-回答对微调模型，使其学会遵循指令并输出结构化回答。

| 参数 | 值 |
|------|-----|
| 训练阶段 | supervised fine-tuning |
| 学习率 | 2e-4（比 CPT 高，信号密度大） |
| epochs | 3 |
| 学习率调度 | cosine decay |
| warmup 比例 | 10% |
| 序列长度 | 2048 tokens |
| 精度 | bf16 |

### 3.4 CPT → SFT 两阶段流水线

先 CPT 注入领域知识（"学知识"），再 SFT 对齐指令跟随能力（"学表达"）。顺序不可反——若先 SFT 再 CPT，CPT 的大量无监督文本会冲淡 SFT 学到的指令跟随能力。

## 4. 训练数据

### 4.1 CPT 语料

| 来源 | 文档数 | 平均 tokens/doc | 估算总 tokens | Token 占比 | 说明 |
|------|--------|----------------|--------------|-----------|------|
| essay_fulltext | 253,569 | 8,645 | ~2.19B | **70.0%** | 食品科学论文全文 |
| fineweb_general | 1,006,172 | 733 | ~737M | 23.5% | 通用英文网页语料 |
| essay_abstract | 433,362 | 415 | ~180M | 5.8% | 食品科学论文摘要 |
| wiki_food | 24,479 | 768 | ~19M | 0.6% | 食品相关维基百科 |
| **总计** | **1,717,582** | — | **~3.12B** | 100% | |

- 按 token 计算：**~76% 食品领域 + ~24% 通用语料**
- 混入通用语料的原因：纯领域 CPT 容易导致灾难性遗忘，混合训练是领域 CPT 的最佳实践
- 去重策略：DOI + 标题元数据去重 → MinHash LSH 文本级去重（Jaccard > 0.8）

### 4.2 SFT 数据

| 项目 | 值 |
|------|-----|

| 格式 | Alpaca format（instruction / input / output） |
| 来源 | 由 Gemini API 从学术论文生成 |
| 语言 | 英文 |

## 5. 分布式训练配置

### 硬件与并行策略

| 项目 | 值 |
|------|-----|
| GPU | 4 × NVIDIA H200 141GB |
| 并行策略 | DDP（数据并行） |
| 集群 | NUS Hopper HPC，PBS 调度器 |

**为什么用 DDP 而非模型并行：** 8B 模型 + LoRA 在单卡仅占 ~20-30GB，H200 有 141GB，完全无需拆分模型。DDP 通信开销极低（LoRA 可训练参数仅 ~100-200M）。

### 有效 Batch Size 计算

```
effective_batch_size = per_device_batch × num_gpus × gradient_accumulation_steps
```

| 实验 | per_device_batch | GPUs | grad_accum | effective_batch |
|------|-----------------|------|------------|----------------|
| CPT（两模型） | 2 | 4 | 8 | **64** |
| SFT Qwen3 | 4 | 4 | 4 | **64** |
| SFT Llama | 2 | 4 | 8 | **64** |

> 注：Llama SFT 的 per_device_batch 降至 2 是因为 Llama hidden_size (4096) > Qwen3 (3840)，batch=4 时 OOM。通过增加 grad_accum 步数补偿，effective batch 保持一致，不影响消融结论。

## 6. 训练结果

### 6.1 CPT 结果

| 模型 | 总步数 | train_loss | 运行时间 | 状态 |
|------|--------|-----------|---------|------|
| Qwen3-8B | 23,773 | ~1.77 | ~52h (含续训) | ✅ 完成 |
| Llama 3.1 8B | 22,953 | **1.777** | 2 天 0h40m | ✅ 完成 |

> 两模型总步数不同（23,773 vs 22,953）是因为不同 tokenizer 对同一语料切分后的 chunk 数不同，属正常现象。

### 6.2 SFT-Only 结果

| 模型 | 总步数 | train_loss | 运行时间 |
|------|--------|-----------|---------|
| Qwen3-8B | 5,754 | **~0.85** | ~5h (含续训) |
| Llama 3.1 8B | 5,754 | **0.984** | 3h39m |

> 不同模型的 loss 不可直接比较（tokenizer 不同，cross-entropy 计算的 token 分布不同）。最终对比需依赖下游评测指标（MCQ Accuracy）。

### 6.3 CPT → SFT 结果
